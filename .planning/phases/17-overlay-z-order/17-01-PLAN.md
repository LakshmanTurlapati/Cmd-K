<objective>
Lower the NSPanel window level from `PanelLevel::Status` (level 25) to `PanelLevel::Floating` (level 3) so that macOS system UI elements — permission dialogs, Notification Center, Spotlight — render above the CMD+K overlay, while preserving the overlay's ability to float above all normal application windows and fullscreen apps.

Purpose: The current `Status` level (25) places the overlay at or above `NSStatusWindowLevel`, which blocks system dialogs and overlays from appearing on top. `Floating` level (3) is the standard level for floating utility panels — it sits above normal windows (level 0) but below modal panels (level 8), the main menu (level 24), and system overlays (level 25+). This is the same approach used by Raycast and similar floating panel apps.

Output: Updated `src-tauri/src/lib.rs` with the correct window level.
</objective>

**Why `Floating` and not `ModalPanel` or a custom value:**
- `Floating` (3) is the standard macOS level for floating utility panels
- It sits above normal windows (0) but below ALL system UI levels
- `ModalPanel` (8) would still block some system dialogs

- The `full_screen_auxiliary()` collection behavior independently handles fullscreen apps — it does not depend on the window level being above `MainMenu` (24)
