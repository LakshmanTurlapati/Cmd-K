  <name>Task 1: Add build-linux job to release.yml</name>
  <files>.github/workflows/release.yml</files>
  <action>
Add a `build-linux` job to release.yml between `build-windows` and `release`. Use a matrix strategy for dual-architecture builds:

Matrix configuration:
```yaml
strategy:
  matrix:
    include:
      - arch: x86_64
        runner: ubuntu-22.04
        rust-target: x86_64-unknown-linux-gnu
      - arch: aarch64
        runner: ubuntu-22.04-arm
        rust-target: aarch64-unknown-linux-gnu
runs-on: ${{ matrix.runner }}
```

Per user decision: native ARM runner (`ubuntu-22.04-arm`), NOT cross-compilation (linuxdeploy cannot cross-compile).

Job steps (follow existing macOS/Windows patterns):
1. Checkout (actions/checkout@v4)
2. Extract version from tag: `echo "VERSION=${GITHUB_REF_NAME#v}" >> $GITHUB_ENV`
3. Install system dependencies via apt-get (no caching per user decision):
   ```bash
   sudo apt-get update
   sudo apt-get install -y \
     libwebkit2gtk-4.1-dev \
     libappindicator3-dev \
     librsvg2-dev \
     patchelf \
     libxdo-dev \
     libssl-dev \
     libdbus-1-dev \
     build-essential \
     curl \
     wget \
     file
   ```
4. Install Rust stable (dtolnay/rust-toolchain@stable)
5. Rust cache (swatinem/rust-cache@v2 with workspaces: src-tauri)
6. Install pnpm (pnpm/action-setup@v4, version: latest)
7. Setup Node.js (actions/setup-node@v4, node-version: 20, cache: pnpm)
8. Install frontend dependencies: `pnpm install`
9. Build Tauri app with env vars:
   - TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
   - TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
   - NO_STRIP: "true" (workaround for linuxdeploy .relr.dyn strip failure on Ubuntu 22.04)
   - APPIMAGE_EXTRACT_AND_RUN: "1" (FUSE not available on CI runners)
   Run: `pnpm tauri build`
10. Rename artifacts (bash step matching Windows pattern):
    ```bash
    APPIMAGE_DIR="src-tauri/target/release/bundle/appimage"
    ORIG_APPIMAGE=$(ls "$APPIMAGE_DIR"/*.AppImage | grep -v '.sig' | head -1)
    ORIG_SIG="${ORIG_APPIMAGE}.sig"
    mv "$ORIG_APPIMAGE" "$APPIMAGE_DIR/CMD+K-${{ env.VERSION }}-linux-${{ matrix.arch }}.AppImage"
    if [ -f "$ORIG_SIG" ]; then
      mv "$ORIG_SIG" "$APPIMAGE_DIR/CMD+K-${{ env.VERSION }}-linux-${{ matrix.arch }}.AppImage.sig"
    fi
    ```
11. Generate checksum:
    ```bash
    cd src-tauri/target/release/bundle/appimage
    sha256sum "CMD+K-${{ env.VERSION }}-linux-${{ matrix.arch }}.AppImage" > "CMD+K-${{ env.VERSION }}-linux-${{ matrix.arch }}.AppImage.sha256"
    ```
12. Upload artifacts (actions/upload-artifact@v4):
    - name: `linux-appimage-${{ matrix.arch }}`
    - path: `src-tauri/target/release/bundle/appimage/CMD+K-*.*`
  </action>

  <name>Task 3: Add Linux write-permission guard to updater</name>
  <files>src-tauri/src/commands/updater.rs</files>
  <action>
Modify `install_pending_update` in updater.rs to check write permissions on Linux before attempting install. Per user decision: if AppImage location is not writable, warn via tray message and skip update (don't attempt write).

Add a Linux-specific guard before the `update.install(&bytes)` call:

```rust
// On Linux (AppImage), check if the executable location is writable
// before attempting install. If not writable, warn and skip.
#[cfg(target_os = "linux")]
{
    if let Ok(exe_path) = std::env::current_exe() {
        if let Some(parent) = exe_path.parent() {
            let test_path = parent.join(".cmd-k-update-test");
            match std::fs::File::create(&test_path) {
                Ok(_) => {
                    let _ = std::fs::remove_file(&test_path);
                }
                Err(e) => {
                    eprintln!(
                        "[updater] AppImage location not writable ({}): {}. Skipping update.",
                        parent.display(),
                        e
                    );
                    // Update tray to inform user
                    if let Ok(mut s) = app.state::<UpdateState>().status.lock() {
                        *s = UpdateStatus::Idle;
                    }
                    update_tray_text(app, &UpdateStatus::Idle);
                    return;
                }
            }
        }
    }
}
```

Place this check inside `install_pending_update`, after extracting `update` and `bytes` from the state but before calling `update.install(&bytes)`. The function signature already takes `app: &tauri::AppHandle`, so state and tray access are available.

Also update the doc comment on `install_pending_update` to mention Linux:
```rust
/// Install the pending update (called from quit handler).
/// On Windows, this triggers the NSIS passive installer which force-exits.
/// On macOS, this replaces the .app bundle.
/// On Linux, this replaces the AppImage file (skips if location is not writable).
```
