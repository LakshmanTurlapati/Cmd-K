  <files>src-tauri/src/terminal/process.rs</files>
  <behavior>
    - get_process_cwd(pid) reads /proc/PID/cwd via read_link, returns Some(path_string) or None on error
    - get_process_name(pid) reads /proc/PID/exe via read_link, strips " (deleted)" suffix, extracts filename
    - get_process_name returns None for permission errors, not panic
    - get_child_pids(pid) reads /proc/PID/task/PID/children, parses space-separated PIDs
    - get_child_pids falls back to scanning /proc/*/stat for ppid match when children file unavailable
    - get_parent_pid(pid) parses /proc/PID/stat, handles comm field with spaces/parens by finding last ')'
    - get_parent_pid returns field index 1 after the closing paren (state=0, ppid=1)
  </behavior>
  <action>
Replace the 4 stub functions with real Linux /proc implementations. Change their cfg gates from `#[cfg(not(any(target_os = "macos", target_os = "windows")))]` to `#[cfg(target_os = "linux")]`. Add a new `#[cfg(target_os = "linux")]` `get_parent_pid` function.

Specific implementations:
1. `get_process_cwd(pid)`: `std::fs::read_link(format!("/proc/{}/cwd", pid))`, convert PathBuf to String, return None on any error. Log errors at eprintln debug level.
2. `get_process_name(pid)`: `std::fs::read_link(format!("/proc/{}/exe", pid))`, trim_end_matches(" (deleted)") on the lossy string, then Path::new(clean).file_name() to extract just the binary name.
3. `get_child_pids(pid)`: Read `/proc/{pid}/task/{pid}/children`, split_whitespace, parse to i32. If that file fails, fall back to scanning all /proc numeric dirs and checking get_parent_pid(child) == pid.
4. `get_parent_pid(pid)`: Read `/proc/{pid}/stat`, find last ')' via rfind, split remainder on whitespace, index [1] is ppid (index [0] is state char). Parse to i32.

All functions: return None/Vec::new() on any IO error. No unwrap, no panic. Use eprintln for debug logging (matching existing codebase pattern).

Write tests using the live /proc filesystem (tests will run on the WSL2 dev environment). Test get_process_cwd/get_process_name/get_parent_pid against the current process (std::process::id()). Test get_child_pids by spawning a child process.
