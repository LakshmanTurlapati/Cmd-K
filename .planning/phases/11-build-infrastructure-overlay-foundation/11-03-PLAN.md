```rust
/// Capture the HWND of the foreground window using Win32 GetForegroundWindow.
///
/// This MUST be called BEFORE showing the overlay because after the overlay
/// appears, GetForegroundWindow returns our own overlay HWND.
#[cfg(target_os = "windows")]
fn get_foreground_hwnd() -> Option<isize> {
    use windows_sys::Win32::UI::WindowsAndMessaging::GetForegroundWindow;
    let hwnd = unsafe { GetForegroundWindow() };
    if hwnd == 0 {
        None
    } else {
        Some(hwnd)
    }
}
```

**Add focus restoration function in hotkey.rs:**

```rust
/// Restore focus to the previously captured HWND using the
/// AttachThreadInput + SetForegroundWindow workaround.
///
/// Windows restricts which processes can call SetForegroundWindow.
/// The workaround: attach our thread's input to the target window's
/// thread, call SetForegroundWindow, then detach.
///
/// Returns true if focus was successfully restored.
#[cfg(target_os = "windows")]
pub fn restore_focus(target_hwnd: isize) -> bool {
    use windows_sys::Win32::UI::WindowsAndMessaging::*;
    use windows_sys::Win32::System::Threading::GetCurrentThreadId;

    unsafe {
        // Validate the HWND is still a valid window (might have been closed)
        if IsWindow(target_hwnd) == 0 {
            eprintln!("[focus] target HWND {} is no longer valid", target_hwnd);
            return false;
        }

        let target_thread = GetWindowThreadProcessId(target_hwnd, std::ptr::null_mut());
        let our_thread = GetCurrentThreadId();

        // Attach our thread's input queue to the target's thread
        let attached = if target_thread != 0 && target_thread != our_thread {
            AttachThreadInput(our_thread, target_thread, 1) != 0 // TRUE = attach
        } else {
            false
        };

        let result = SetForegroundWindow(target_hwnd);

        // Always detach if we attached
        if attached {
            AttachThreadInput(our_thread, target_thread, 0); // FALSE = detach
        }

        if result != 0 {
            eprintln!("[focus] SetForegroundWindow succeeded for HWND {}", target_hwnd);
            true
        } else {
            eprintln!("[focus] SetForegroundWindow failed for HWND {}, trying fallback", target_hwnd);
            // Fallback: AllowSetForegroundWindow then retry
            // This handles edge cases where AttachThreadInput alone is insufficient
            let mut target_pid: u32 = 0;
            GetWindowThreadProcessId(target_hwnd, &mut target_pid);
            AllowSetForegroundWindow(target_pid);
            let retry = SetForegroundWindow(target_hwnd);
            eprintln!("[focus] Fallback SetForegroundWindow result: {}", retry != 0);
            retry != 0
        }
    }
}

#[cfg(not(target_os = "windows"))]
pub fn restore_focus(_target_hwnd: isize) -> bool {
    false
}
```

**Click-outside dismiss:** The frontend currently handles click-outside by detecting window blur and calling `hide_overlay`. On Windows, this should work the same way -- when the user clicks outside, the Tauri window loses focus, frontend detects blur, calls hide_overlay IPC, which hides the window and restores focus. The focus restoration in hide_overlay handles this correctly because clicking outside means focus went to another window (which is now the foreground), so we actually want to let that window stay focused. To handle this correctly:

- If overlay was dismissed by hotkey re-press or Escape: restore focus to previous_hwnd.
- If overlay was dismissed by click-outside: the user clicked on another window, which is now foreground. We should NOT forcefully restore focus to the original previous_hwnd. Instead, just hide the window.

To distinguish these cases, add a `restore_focus` parameter to hide_overlay:

```rust
#[tauri::command]

    // ... hide the window ...

    #[cfg(target_os = "windows")]
    {
        if restore_focus_on_hide {
            // Restore focus to previously captured window
            // (called from hotkey toggle or Escape)
        }
        // If !restore_focus_on_hide, user clicked elsewhere -- let that window keep focus
    }
}
```

WAIT -- changing the hide_overlay signature would break existing IPC calls from the frontend. Instead, create a separate internal function for focus-restoring hide, and keep the IPC command signature unchanged. The IPC hide_overlay (called from frontend on blur/Escape) should NOT restore focus on click-outside, but SHOULD on Escape.

Actually, simpler approach: The frontend already distinguishes Escape (explicit dismiss) from blur (click-outside). On Escape, the frontend calls `hide_overlay`. On blur, the frontend also calls `hide_overlay`. But on Windows, click-outside means the user clicked another window which is now foreground -- restoring focus would fight the user's intent.

Solution: Always restore focus in hide_overlay. If the user clicked outside, the click already gave focus to the new window. Calling SetForegroundWindow on the ORIGINAL previous_hwnd would be wrong. Instead, in hide_overlay, check if the current foreground window IS our overlay -- if so, restore focus. If not (user already clicked elsewhere), skip restoration.

```rust
#[cfg(target_os = "windows")]
{
    // Only restore focus if the overlay is still the foreground window.
    // If user clicked outside, the clicked window is already foreground.
    use windows_sys::Win32::UI::WindowsAndMessaging::GetForegroundWindow;
    let current_fg = unsafe { GetForegroundWindow() };
    let our_hwnd = /* get our window HWND */;

    if let Some(state) = app.try_state::<AppState>() {
        let prev_hwnd = state.previous_hwnd.lock().ok().and_then(|g| *g);
        if let Some(prev) = prev_hwnd {
            // Check if we're still the foreground window (Escape/hotkey dismiss)
            // vs user clicked outside (another window is already foreground)
            let we_are_foreground = current_fg == our_hwnd;
            if we_are_foreground {
                let restored = crate::commands::hotkey::restore_focus(prev);
                eprintln!("[hide_overlay] focus restored to HWND {}: {}", prev, restored);
            } else {
                eprintln!("[hide_overlay] not restoring focus -- user clicked outside");
            }
        }
    }
}
```
