  <action>
In the hotkey handler's `if !is_currently_visible` block (the "about to show" branch), restructure the code so the Windows HWND capture executes INDEPENDENTLY of the macOS PID capture result.

Currently (broken structure):
```
if !is_currently_visible {
    let pid = get_frontmost_pid();           // Always None on Windows
    if let Some(pid) = pid {                 // Never matches on Windows
        // ... macOS PID storage ...
        #[cfg(target_os = "windows")]
        { get_foreground_hwnd() ... }        // UNREACHABLE on Windows
        // ... other macOS-specific code ...
    }
}
```

Fix to (correct structure):
```
if !is_currently_visible {
    // Windows: capture HWND BEFORE overlay steals focus (independent of PID)
    #[cfg(target_os = "windows")]
    {
        let hwnd = get_foreground_hwnd();
        eprintln!("[hotkey] Windows: capturing foreground HWND: {:?}", hwnd);
        if let Some(state) = app_handle.try_state::<AppState>() {
            if let Ok(mut prev) = state.previous_hwnd.lock() {
                *prev = hwnd;
            }
        }
    }

    let pid = get_frontmost_pid();
    if let Some(pid) = pid {
        // ... macOS PID storage (unchanged) ...
        // REMOVE the #[cfg(target_os = "windows")] HWND block from here
        // ... rest of macOS-specific code (unchanged) ...
    }
}
```

Key points:
- The `#[cfg(target_os = "windows")]` HWND capture block moves to BEFORE `let pid = get_frontmost_pid()` and OUTSIDE `if let Some(pid) = pid`
- It sits at the `if !is_currently_visible` scope level so it always executes on Windows when the overlay is about to show
- The HWND capture MUST happen before toggle_overlay() because after the overlay shows, GetForegroundWindow returns our overlay's HWND
- All other code inside `if let Some(pid) = pid` remains UNCHANGED (macOS PID storage, AX text capture, IDE CWD capture, window key computation)
- The HWND clear in the `else` branch (overlay visible, hiding) at lines 338-346 is ALREADY correctly placed and needs no changes
- Do NOT modify window.rs -- the focus restoration code there is already correct and will work once previous_hwnd is populated
