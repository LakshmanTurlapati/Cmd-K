  <name>Task 1: Parameterize build-dmg.sh for CI compatibility</name>
  <files>scripts/build-dmg.sh</files>
  <action>
Modify the variable declarations at the top of `scripts/build-dmg.sh` to read from environment variables with local-build fallbacks:

1. **VERSION**: Replace `VERSION="0.2.2"` with logic that:
   - First checks `$VERSION` env var (set by CI from tag)
   - Falls back to extracting version from `src-tauri/tauri.conf.json` using grep + sed (no jq dependency)
   - Example: `VERSION="${VERSION:-$(grep '"version"' "$PROJECT_ROOT/src-tauri/tauri.conf.json" | head -1 | sed 's/.*: *"\(.*\)".*/\1/')}"`
   - This fixes the 0.2.2 vs 0.2.4 mismatch AND makes it CI-compatible

2. **SIGNING_IDENTITY**: Replace hardcoded value with:
   `SIGNING_IDENTITY="${SIGNING_IDENTITY:-"Developer ID Application: VENKAT LUKSSHMAN TURLAPATI (36L722DZ7X)"}"`
   - Local: uses default. CI: passed via env var (same value, but configurable)

3. **KEYCHAIN_PROFILE**: Replace hardcoded value with:
   `KEYCHAIN_PROFILE="${KEYCHAIN_PROFILE:-CMD-K-NOTARIZE}"`
   - Local: uses CMD-K-NOTARIZE (existing profile). CI: workflow sets this to the temp keychain profile name

4. **Notarization command**: The `xcrun notarytool submit` call currently uses `--keychain-profile "$KEYCHAIN_PROFILE"`. Add an alternative path that uses `--apple-id` / `--team-id` / `--password` when `$APPLE_ID` env var is set:
   ```bash
   if [ -n "${APPLE_ID:-}" ]; then
     # CI path: use explicit credentials
     NOTARIZE_OUTPUT=$(xcrun notarytool submit "$DMG_PATH" \
       --apple-id "$APPLE_ID" \
       --team-id "$APPLE_TEAM_ID" \
       --password "$APPLE_APP_PASSWORD" \
       --wait 2>&1) || true
   else
     # Local path: use stored keychain profile
     NOTARIZE_OUTPUT=$(xcrun notarytool submit "$DMG_PATH" \
       --keychain-profile "$KEYCHAIN_PROFILE" \
       --wait 2>&1) || true
   fi
   ```
   Apply the same pattern to the `xcrun notarytool log` call in the error handler.

5. **DMG styling AppleScript**: The `osascript` block that styles the DMG window with Finder uses GUI commands that require a display. In CI (headless), this will fail. Wrap the entire osascript block and the background image copy in a conditional:
   ```bash
   if [ "${CI:-}" != "true" ]; then
     # Copy background image and run AppleScript styling
     # ... existing code ...
   else
     echo "  Skipping DMG window styling (CI environment)"
   fi
   ```
   The DMG will still work fine without styling -- it just won't have the pretty drag-to-Applications layout.

6. **Keep all other behavior identical.** The preflight checks, build command, code signing, verification, and stapling steps remain unchanged. Local builds should see zero behavioral difference.

Do NOT change APP_NAME, IDENTIFIER, or TARGET -- those are genuinely constant.

**Section 1: Required GitHub Secrets**
Table of all 5 secrets with name, description, and where to get each:
| Secret | Description | Source |
|--------|-------------|--------|
| APPLE_CERTIFICATE_BASE64 | Base64-encoded .p12 Developer ID certificate | Keychain Access export |
| APPLE_CERTIFICATE_PASSWORD | Password for the .p12 file | Set during export |
| APPLE_ID | Apple ID email for notarization | lakshmantvnm@gmail.com |
| APPLE_TEAM_ID | Apple Developer Team ID | 36L722DZ7X |
| APPLE_APP_PASSWORD | App-specific password for notarization | appleid.apple.com |

**Section 2: Export Developer ID Certificate (.p12)**
Step-by-step with exact UI paths:
1. Open Keychain Access
2. Select "login" keychain in sidebar
3. Category: "My Certificates"
4. Find "Developer ID Application: VENKAT LUKSSHMAN TURLAPATI (36L722DZ7X)"
5. Right-click > Export "Developer ID Application..."
6. Save as .p12 format, set a strong password
7. Base64 encode: `base64 -i certificate.p12 | pbcopy`
8. The clipboard now contains the value for APPLE_CERTIFICATE_BASE64

**Section 3: Generate App-Specific Password**
1. Go to appleid.apple.com > Sign-In and Security > App-Specific Passwords
2. Generate a new password, label it "CMD-K CI Notarization"
3. Copy the generated password -- this is the APPLE_APP_PASSWORD value

**Section 4: Add Secrets to GitHub**
1. Go to GitHub repo > Settings > Secrets and variables > Actions
2. Click "New repository secret" for each:
   - APPLE_CERTIFICATE_BASE64: paste the base64 string
   - APPLE_CERTIFICATE_PASSWORD: the password from step 6 above
   - APPLE_ID: lakshmantvnm@gmail.com
   - APPLE_TEAM_ID: 36L722DZ7X
   - APPLE_APP_PASSWORD: the app-specific password

**Section 5: Windows Signing (Future)**

- WINDOWS_CERTIFICATE: base64-encoded .pfx
- WINDOWS_CERTIFICATE_PASSWORD: pfx password
The release workflow has a conditional block that activates when these secrets are present.

**Section 6: Verification**
How to verify secrets are working: push a `v*` tag and check the Actions tab.
