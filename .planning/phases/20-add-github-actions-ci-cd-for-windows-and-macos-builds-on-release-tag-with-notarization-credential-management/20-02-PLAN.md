Create `.github/workflows/release.yml`. Must create `.github/workflows/` directory first.

The workflow has 3 jobs: `build-macos`, `build-windows`, and `release`.

**Trigger:**
```yaml
on:
  push:
    tags:
      - 'v*'
```

**Job 1: build-macos** (runs-on: macos-latest)

Steps:
1. **Checkout**: `actions/checkout@v4`

2. **Extract version from tag**: Strip `v` prefix from `GITHUB_REF_NAME`:
   ```yaml
   - name: Extract version from tag
     run: echo "VERSION=${GITHUB_REF_NAME#v}" >> $GITHUB_ENV
   ```

3. **Install Rust stable**: `dtolnay/rust-toolchain@stable`

4. **Add Rust targets**: `rustup target add x86_64-apple-darwin aarch64-apple-darwin`

5. **Rust cache**: `swatinem/rust-cache@v2` with `workspaces: src-tauri`

6. **Install pnpm**: `pnpm/action-setup@v4` with `version: latest`

7. **Setup Node.js**: `actions/setup-node@v4` with `node-version: 20`, `cache: pnpm`

8. **Install frontend deps**: `pnpm install`

9. **Import code signing certificate**: Create temporary keychain, import p12:
   ```yaml
   - name: Import code signing certificate
     env:
       APPLE_CERTIFICATE_BASE64: ${{ secrets.APPLE_CERTIFICATE_BASE64 }}
       APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
     run: |
       CERTIFICATE_PATH=$RUNNER_TEMP/certificate.p12
       KEYCHAIN_PATH=$RUNNER_TEMP/app-signing.keychain-db
       KEYCHAIN_PASSWORD=$(openssl rand -base64 32)

       # Decode certificate
       echo "$APPLE_CERTIFICATE_BASE64" | base64 --decode > "$CERTIFICATE_PATH"

       # Create temporary keychain
       security create-keychain -p "$KEYCHAIN_PASSWORD" "$KEYCHAIN_PATH"
       security set-keychain-settings -lut 21600 "$KEYCHAIN_PATH"
       security unlock-keychain -p "$KEYCHAIN_PASSWORD" "$KEYCHAIN_PATH"

       # Import certificate
       security import "$CERTIFICATE_PATH" -P "$APPLE_CERTIFICATE_PASSWORD" \
         -A -t cert -f pkcs12 -k "$KEYCHAIN_PATH"
       security set-key-partition-list -S apple-tool:,apple: -k "$KEYCHAIN_PASSWORD" "$KEYCHAIN_PATH"

       # Add to search list (prepend so it's found first)
       security list-keychains -d user -s "$KEYCHAIN_PATH" $(security list-keychains -d user | tr -d '"')

       # Store notarization credentials in the temp keychain
       xcrun notarytool store-credentials "CI-NOTARIZE" \
         --apple-id "${{ secrets.APPLE_ID }}" \
         --team-id "${{ secrets.APPLE_TEAM_ID }}" \
         --password "${{ secrets.APPLE_APP_PASSWORD }}" \
         --keychain "$KEYCHAIN_PATH"
   ```

10. **Build signed + notarized DMG**: Run the parameterized build-dmg.sh:
    ```yaml
    - name: Build signed and notarized DMG
      env:
        VERSION: ${{ env.VERSION }}
        SIGNING_IDENTITY: "Developer ID Application: VENKAT LUKSSHMAN TURLAPATI (36L722DZ7X)"
        KEYCHAIN_PROFILE: "CI-NOTARIZE"
        APPLE_ID: ${{ secrets.APPLE_ID }}
        APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
        APPLE_APP_PASSWORD: ${{ secrets.APPLE_APP_PASSWORD }}
        CI: "true"
      run: chmod +x scripts/build-dmg.sh && ./scripts/build-dmg.sh
    ```

11. **Rename DMG to standard name**: The build-dmg.sh outputs `CMD+K-{version}-universal.dmg` in the `release/` directory. Rename to ensure consistent naming:
    ```yaml
    - name: Rename artifact
      run: |
        mv "release/CMD+K-${{ env.VERSION }}-universal.dmg" "release/CMD+K-${{ env.VERSION }}-universal.dmg" || true
    ```
    (This is effectively a no-op since build-dmg.sh already uses the right name with $VERSION, but it serves as a safety net)

12. **Generate SHA256 checksum**:
    ```yaml
    - name: Generate checksum
      run: |
        cd release
        shasum -a 256 "CMD+K-${{ env.VERSION }}-universal.dmg" > "CMD+K-${{ env.VERSION }}-universal.dmg.sha256"
    ```

13. **Upload artifacts**: `actions/upload-artifact@v4` with name `macos-dmg` and path `release/CMD+K-*.dmg*`

14. **Cleanup keychain**:
    ```yaml
    - name: Cleanup keychain
      if: always()
      run: security delete-keychain $RUNNER_TEMP/app-signing.keychain-db 2>/dev/null || true
    ```

**Job 2: build-windows** (runs-on: windows-latest)

Steps:
1. **Checkout**: `actions/checkout@v4`

2. **Extract version from tag**:
   ```yaml
   - name: Extract version from tag
     shell: bash
     run: echo "VERSION=${GITHUB_REF_NAME#v}" >> $GITHUB_ENV
   ```

3. **Install Rust stable**: `dtolnay/rust-toolchain@stable`

4. **Rust cache**: `swatinem/rust-cache@v2` with `workspaces: src-tauri`

5. **Install pnpm**: `pnpm/action-setup@v4` with `version: latest`

6. **Setup Node.js**: `actions/setup-node@v4` with `node-version: 20`, `cache: pnpm`

7. **Install frontend deps**: `pnpm install`

8. **Build Tauri app**: `pnpm tauri build` (produces NSIS installer)

9. **Conditional Windows signing** (skipped when secrets not set):
   ```yaml
   - name: Sign Windows installer
     if: env.WINDOWS_CERTIFICATE != ''
     env:
       WINDOWS_CERTIFICATE: ${{ secrets.WINDOWS_CERTIFICATE }}
       WINDOWS_CERTIFICATE_PASSWORD: ${{ secrets.WINDOWS_CERTIFICATE_PASSWORD }}
     shell: pwsh
     run: |
       $pfxBytes = [Convert]::FromBase64String($env:WINDOWS_CERTIFICATE)
       $pfxPath = "$env:RUNNER_TEMP\certificate.pfx"
       [IO.File]::WriteAllBytes($pfxPath, $pfxBytes)
       Import-PfxCertificate -FilePath $pfxPath -CertStoreLocation Cert:\CurrentUser\My `
         -Password (ConvertTo-SecureString -String $env:WINDOWS_CERTIFICATE_PASSWORD -AsPlainText -Force)
       $exe = Get-ChildItem -Path "src-tauri\target\release\bundle\nsis\*.exe" | Select-Object -First 1
       & "C:\Program Files (x86)\Windows Kits\10\bin\10.0.22621.0\x64\signtool.exe" `
         sign /fd SHA256 /tr http://timestamp.digicert.com /td SHA256 /a $exe.FullName
   ```

10. **Rename NSIS installer** to standard name:
    ```yaml
    - name: Rename artifact
      shell: bash
      run: |
        NSIS_DIR="src-tauri/target/release/bundle/nsis"
        ORIG=$(ls "$NSIS_DIR"/*.exe | head -1)
        mv "$ORIG" "$NSIS_DIR/CMD+K-${{ env.VERSION }}-windows-x64.exe"
    ```

11. **Generate SHA256 checksum**:
    ```yaml
    - name: Generate checksum
      shell: bash
      run: |
        cd src-tauri/target/release/bundle/nsis
        sha256sum "CMD+K-${{ env.VERSION }}-windows-x64.exe" > "CMD+K-${{ env.VERSION }}-windows-x64.exe.sha256"
    ```

12. **Upload artifacts**: `actions/upload-artifact@v4` with name `windows-installer` and path `src-tauri/target/release/bundle/nsis/CMD+K-*.*`

**Job 3: release** (needs: [build-macos, build-windows])

Steps:
1. **Download all artifacts**: `actions/download-artifact@v4` with `path: artifacts`

2. **Create GitHub Release**: Use `softprops/action-gh-release@v2`:
   ```yaml
   - name: Create GitHub Release
     uses: softprops/action-gh-release@v2
     with:
       generate_release_notes: false
       body: |
         ## CMD+K ${{ github.ref_name }}

         ### Downloads

         | Platform | File |
         |----------|------|
         | macOS (Universal) | CMD+K-${{ env.VERSION }}-universal.dmg |
         | Windows (x64) | CMD+K-${{ env.VERSION }}-windows-x64.exe |

         ### Checksums (SHA256)

         See the `.sha256` files attached to this release.

         ---
         _Edit this release to add detailed release notes._
       files: |
         artifacts/macos-dmg/*
         artifacts/windows-installer/*
     env:
       GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
   ```

**Important details:**
- Do NOT use `tauri-apps/tauri-action` -- use raw build commands per user's discretion note
- Use `pnpm/action-setup@v4` (not v2/v3)
- The macOS job uses the full build-dmg.sh pipeline (build + sign + notarize + staple + DMG)
- The Windows job uses `pnpm tauri build` directly (build steps inlined per user decision)
- Add `permissions: contents: write` at the workflow level for release creation
- The version extraction step uses `${GITHUB_REF_NAME#v}` in bash to strip the `v` prefix
