Edit src/styles.css scrollbar section (lines 155-175). The existing code has:
- scrollbar-color on * selector (line 157)
- ::-webkit-scrollbar pseudo-elements (lines 160-175)

The problem: macOS uses overlay scrollbars by default, which override webkit-scrollbar pseudo-elements in WebKit/Chromium. The fix is to add `scrollbar-width: thin` to the global `*` selector alongside the existing `scrollbar-color`. This uses the standard CSS Scrollbars API which macOS respects.

Replace the scrollbar section (lines 155-175) with:

```css
/* Scrollbar styling — thin track, subtle thumb (cross-platform) */
* {
  scrollbar-width: thin;
  scrollbar-color: rgba(255, 255, 255, 0.15) transparent;
}

*::-webkit-scrollbar {
  width: 6px;
}

*::-webkit-scrollbar-track {
  background: transparent;
}

*::-webkit-scrollbar-thumb {
  background: rgba(255, 255, 255, 0.15);
  border-radius: 3px;
}

*::-webkit-scrollbar-thumb:hover {
  background: rgba(255, 255, 255, 0.25);
}
```

Key change: Added `scrollbar-width: thin` on line 2 of the `*` block. This is the standard CSS property that forces the browser to use thin scrollbars on all platforms. On macOS, this overrides the system overlay scrollbar behavior. The webkit-scrollbar pseudo-elements remain as fallback for older engines.

Do NOT modify any component files -- the `scrollbar-thin` Tailwind class they use also sets `scrollbar-width: thin`, which is now consistent with the global rule.
