**Regex pattern guidelines:**
- Use `\b` word boundaries to prevent false positives on substrings
- Use `(?i)` for case-insensitive patterns where appropriate (Windows commands, SQL)
- Use `\s+` for required whitespace between command and arguments
- For config file redirects, match the `>` redirect operator followed by the path
- Do NOT use `(?i)` for Unix commands (they are case-sensitive)
- Keep Windows patterns case-insensitive with `(?i)`
