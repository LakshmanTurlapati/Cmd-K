**linux_reader.rs:** Create new module at `src-tauri/src/terminal/linux_reader.rs` paralleling `ax_reader.rs` (macOS) and `uia_reader.rs` (Windows). Structure:

1. **`read_terminal_text_linux(pid: i32, exe_name: &str) -> Option<String>`** (pub fn, cfg(target_os = "linux")):
   - Match `exe_name` to determine strategy:
     - `"gnome-terminal-server" | "tilix" | "terminator" | "mate-terminal" | "xfce4-terminal" | "guake" | "tilda" | "sakura" | "lxterminal" | "terminology" | "konsole"` -> `read_via_atspi(pid)`
     - `"kitty"` -> `read_via_kitty()`
     - `"wezterm-gui"` -> `read_via_wezterm()`
     - `_` -> `None` (graceful degradation for alacritty, st, foot, xterm, urxvt, etc.)

2. **Non-Linux stub** (cfg(not(target_os = "linux"))):
   - `pub fn read_terminal_text_linux(_pid: i32, _exe_name: &str) -> Option<String> { None }`

3. **`read_via_atspi(pid: i32) -> Option<String>`** (internal, cfg(target_os = "linux")):
   - Connect to session bus via `zbus::blocking::Connection::session()`
   - Call `org.a11y.Bus.GetAddress` on the session bus to get the AT-SPI2 bus address
   - Connect to the AT-SPI2 bus using `zbus::blocking::Connection::builder().address(addr).build()`
   - Get desktop children from registry: call `GetChildren` on `org.a11y.atspi.Accessible` interface at `org.a11y.atspi.Registry` bus name, path `/org/a11y/atspi/accessible/root`
   - For each child (application): get its PID via property on `org.a11y.atspi.Application` interface, compare to target `pid`
   - Once matching app found: walk its children looking for `GetRole() == 60` (Terminal role constant)
   - On the terminal widget: read `CharacterCount` property from `org.a11y.atspi.Text` interface, then call `GetText(0, count)` to get visible text
   - Wrap entire flow in `std::thread::spawn` + `recv_timeout(Duration::from_millis(500))` for 500ms timeout
   - Use `?` operator with Option chaining throughout -- any failure returns None silently
   - Log with eprintln at entry point only (e.g., "[linux_reader] attempting AT-SPI2 for pid={}")

4. **`read_via_kitty() -> Option<String>`** (internal, cfg(target_os = "linux")):
   - Spawn `Command::new("kitty").args(["@", "get-text", "--extent", "screen"])`
   - Use thread+channel pattern with 500ms timeout (same as AT-SPI2)
   - Check `output.status.success()` -- if false, return None (remote control likely disabled)
   - Convert stdout to String via `String::from_utf8_lossy`, return None if empty after trim

5. **`read_via_wezterm() -> Option<String>`** (internal, cfg(target_os = "linux")):
   - Spawn `Command::new("wezterm").args(["cli", "get-text"])`
   - Same thread+channel 500ms timeout pattern
   - Check success, convert to String, return None if empty

**Key implementation notes:**
- The AT-SPI2 D-Bus calls use zbus `call_method` directly -- do NOT use the `atspi` crate (async-only, heavy deps)
- For AT-SPI2 bus address: `zbus::address::Address::try_from(addr_string)` may be needed to parse the address string before passing to builder
- `GetChildren` on AT-SPI2 returns `Vec<(String, OwnedObjectPath)>` -- these are `(bus_name, object_path)` pairs
- For PID matching: call `org.freedesktop.DBus.Properties.Get("org.a11y.atspi.Application", "Id")` on the app's accessible -- this returns the PID as `i32`
- The `CharacterCount` property is read via `org.freedesktop.DBus.Properties.Get("org.a11y.atspi.Text", "CharacterCount")` returning `i32`
- If AT-SPI2 bus is not running (ServiceUnknown error), return None immediately -- no logging spam
- Thread timeout pattern: `let (tx, rx) = mpsc::channel(); thread::spawn(move || { tx.send(do_work()) }); rx.recv_timeout(Duration::from_millis(500)).ok()?.ok()?`
