**Create src-tauri/src/commands/usage.rs:**

1. Define `UsageStatEntry` as a Serialize struct with fields:
   - `provider: String`
   - `model: String`
   - `input_tokens: u64`
   - `output_tokens: u64`
   - `query_count: u32`
   - `estimated_cost: Option<f64>` (None when pricing unavailable)
   - `pricing_available: bool`

2. Define `UsageStatsResponse` as a Serialize struct with fields:
   - `entries: Vec<UsageStatEntry>`
   - `session_total_cost: Option<f64>` (sum of all entries with known pricing; None if zero entries have pricing)

3. Implement `#[tauri::command] pub fn get_usage_stats(state: tauri::State<'_, crate::state::AppState>) -> UsageStatsResponse`:
   - Lock `state.usage` and iterate over entries
   - For each `(provider_name, model_id) -> UsageEntry`, look up pricing:
     a. First check curated models: call `super::models::curated_models_pricing()` (a new helper -- see below) to get a `HashMap<String, (f64, f64)>` mapping model_id -> (input_price_per_m, output_price_per_m)
     b. If not found in curated, check `state.openrouter_pricing` (lock it) for the model_id
     c. If pricing found, calculate: `cost = (input_tokens as f64 * input_price / 1_000_000.0) + (output_tokens as f64 * output_price / 1_000_000.0)`
     d. If no pricing, set `estimated_cost: None, pricing_available: false`
   - Compute `session_total_cost`: sum all entries where `estimated_cost.is_some()`. If no entries have pricing, return None.
   - Return `UsageStatsResponse`

4. Implement `#[tauri::command] pub fn reset_usage(state: tauri::State<'_, crate::state::AppState>)`:
   - Lock `state.usage` and call `acc.reset()`
   - No return value needed (returns `()`)

**In models.rs -- add pricing lookup helper:**

1. Make `curated_models` function `pub(crate)` (change from private `fn` to `pub(crate) fn`).

2. Add a helper function `pub(crate) fn curated_models_pricing() -> HashMap<String, (f64, f64)>`:
   - Iterate over ALL providers (OpenAI, Anthropic, Gemini, XAI) calling `curated_models(&provider)`
   - Collect into a HashMap where key is model id, value is `(input_price_per_m.unwrap_or(0.0), output_price_per_m.unwrap_or(0.0))`
   - Only include entries where both prices are Some
   - Import `HashMap` from std::collections

Add `pub mod usage;`

**In lib.rs:**

1. Add import: `commands::usage::{get_usage_stats, reset_usage}`
2. Add both commands to `tauri::generate_handler![...]` list

<task type="auto">
  <name>Task 2: Fetch and cache OpenRouter pricing from /api/v1/models response</name>
  <files>src-tauri/src/commands/models.rs</files>
  <action>
**In models.rs:**

1. Extend `OpenRouterModel` deserialize struct to include pricing fields:
   ```rust
   #[derive(Deserialize)]
   struct OpenRouterModel {
       id: String,
       context_length: Option<u64>,
       pricing: Option<OpenRouterPricing>,
   }

   #[derive(Deserialize)]
   struct OpenRouterPricing {
       prompt: Option<String>,      // USD per token as string, e.g. "0.000003"
       completion: Option<String>,  // USD per token as string
   }
   ```

2. Modify `fetch_models` to accept `state: tauri::State<'_, crate::state::AppState>` as an additional parameter. This requires updating the function signature and the Tauri command registration (already in lib.rs generate_handler).

3. In the OpenRouter branch of `fetch_api_models` (or better: move the OpenRouter pricing caching into `fetch_models` itself since that's the command that has access to state), after parsing the OpenRouter response:
   - Build a `HashMap<String, (f64, f64)>` from the parsed models:
     - For each model with `pricing.prompt` and `pricing.completion` both Some:
       - Parse the string values to f64
       - Multiply by 1,000,000 to convert from per-token to per-million-token format
       - Insert into the map: `(model_id, (input_per_m, output_per_m))`
   - Lock `state.openrouter_pricing` and replace the entire HashMap with the new one

4. For OpenRouter models that have pricing, also set `input_price_per_m` and `output_price_per_m` on the returned `ModelWithMeta` structs (so the frontend can also show per-model pricing if desired).

5. For non-OpenRouter providers, `fetch_models` doesn't need to touch openrouter_pricing -- they use curated pricing from `curated_models()`.

**Important edge cases:**
- OpenRouter pricing strings may be "0" for free models -- parse as 0.0, which is valid pricing (free)
- If pricing parse fails (malformed string), skip that model's pricing (treat as None)
- If the API request fails entirely, openrouter_pricing retains its previous value (don't clear on error)

**Note:** Since `fetch_models` now takes `state`, and it's a `#[tauri::command]`, Tauri automatically injects it. No frontend JS changes needed -- Tauri State injection is transparent to the caller.
