Create `src-tauri/src/terminal/context.rs` implementing the full smart context pipeline as pure functions. Follow TDD: write all tests first (RED), then implement (GREEN).

**Constants:**
- `TERMINAL_BUDGET_FRACTION: f64 = 0.12` (12% of context window)
- `DEFAULT_CONTEXT_WINDOW: u32 = 128_000` (128K tokens for unknown models)
- `CHARS_PER_TOKEN: u32 = 4`

**Lazy static regexes** (follow `filter.rs` pattern with `once_cell::sync::Lazy`):
- `ANSI_RE`: Matches CSI sequences (including DEC private `\x1b[?...`), OSC sequences (BEL or ST terminated), charset selection (`\x1b(` / `\x1b)`), simple ESC sequences. Use the comprehensive regex from RESEARCH.md.
- `CONTROL_RE`: Matches `[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]` (all control chars except \n and \t)
- `PROMPT_RE`: Matches shell prompt patterns at line start -- `user@host:path$`, `PS C:\path>`, `C:\path>`, generic prompt chars, Python REPL `>>>`. Use multiline mode (`(?m)`) so `^` matches line starts.

**Internal struct:**
```rust
struct CommandSegment {
    prompt_and_command: String,
    output: String,
}
```

**Private functions:**
- `strip_ansi_and_control(text: &str) -> String`: Apply ANSI_RE then CONTROL_RE, also normalize `\r\n` to `\n` first
- `compute_budget_chars(context_window: u32) -> usize`: `(cw * 0.12 * 4)` as usize, with 0-fallback to DEFAULT
- `segment_commands(text: &str) -> Vec<CommandSegment>`: Split text by PROMPT_RE matches. Lines before first prompt go into a segment with empty prompt_and_command. Collapse consecutive prompt-only lines (empty segments pitfall).
- `smart_truncate(text: &str, budget_chars: usize) -> String`: Fast path if fits. Segment, then build from newest to oldest. If single segment overflows, tail-truncate output at line boundary. If no segments/no prompts, simple tail truncation at line boundary.

**Public functions:**
- `pub fn prepare_terminal_context(raw_text: &str, context_window: u32) -> String`: Pipeline: normalize CRLF -> strip ANSI+control -> compute budget -> smart truncate. Return empty string for empty input.
- `pub fn context_window_for_model(model_id: &str) -> u32`: Prefix-based lookup from static list. Use the model family prefixes from RESEARCH.md code example. Return DEFAULT_CONTEXT_WINDOW for unrecognized models.

**Important details:**
- No `#[cfg(target_os)]` anywhere in this file (SCTX-04 cross-platform requirement)
- Use `once_cell::sync::Lazy` (not `lazy_static!`) to match project convention in filter.rs
- `context_window_for_model` is `pub(crate)` since ai.rs needs it but it's not a public API
- `prepare_terminal_context` is `pub` for the module's external interface
