**providers/mod.rs:**
- `Provider` enum with variants: OpenAI, Anthropic, Gemini, XAI, OpenRouter. Derive Debug, Clone, Serialize, Deserialize, PartialEq. Use `#[serde(rename_all = "snake_case")]`.
- Implement `Provider` methods:
  - `keychain_account(&self) -> &str` -- returns "openai_api_key", "anthropic_api_key", "gemini_api_key", "xai_api_key", "openrouter_api_key"
  - `base_url(&self) -> &str` -- returns the API base URL for each provider (see RESEARCH.md endpoint table)
  - `display_name(&self) -> &str` -- returns "OpenAI", "Anthropic", "Google Gemini", "xAI", "OpenRouter"
  - `timeout_secs(&self) -> u64` -- xAI: 10, all others: 30
  - `adapter(&self) -> StreamAdapter` -- OpenAI/XAI/OpenRouter -> OpenAICompat, Anthropic -> Anthropic, Gemini -> Gemini
  - `from_str(s: &str) -> Result<Self, String>` -- parse "openai", "anthropic", "gemini", "xai", "openrouter" (case-insensitive). Return descriptive error for unknown provider.
- `StreamAdapter` enum with variants: OpenAICompat, Anthropic, Gemini
- `ModelWithMeta` struct (replaces XaiModelWithMeta): `id: String`, `label: String`, `tier: String`. Derive Serialize, Clone.
- `ChatMessage` struct: `role: String`, `content: String`. Derive Deserialize. (Move from ai.rs)
- `default_models(provider: &Provider) -> Vec<ModelWithMeta>` -- hardcoded curated model lists per provider (see RESEARCH.md code example). OpenRouter returns empty vec (always API-fetched).
- `migrate_v024_key(app: &tauri::AppHandle) -> Result<(), String>` -- reads old "xai_api_key" from keychain. If found, the xAI provider already uses the same account name "xai_api_key" so no copy needed -- just set provider to "xai" and default model to "grok-3" in settings.json via tauri-plugin-store. Record `"migration_v026_done": true` in settings.json to prevent re-runs. If no old key found, skip silently.

Re-export `openai_compat`, `anthropic`, `gemini` submodules.

**providers/openai_compat.rs:**
- `pub async fn stream(provider: &Provider, api_key: &str, model: &str, system_prompt: String, user_message: String, history: Vec<ChatMessage>, on_token: tauri::ipc::Channel<String>) -> Result<(), String>`
- Build messages array: system role first, then history, then user message. Include `"stream": true, "temperature": 0.1`.
- POST to `{provider.base_url()}/chat/completions` with `Authorization: Bearer {api_key}`.
- For OpenRouter specifically: add `HTTP-Referer: https://github.com/cmd-k-app` and `X-Title: CMD+K` headers.
- Check HTTP status: 200 proceed, 401 -> "{display_name}: Authentication failed. Check your API key.", 429 -> "{display_name}: Rate limited. Wait a moment and try again.", other -> "{display_name}: API error ({status}). Try again."
- Parse SSE with `response.bytes_stream().eventsource()` wrapped in `tokio::time::timeout(Duration::from_secs(provider.timeout_secs()))`.
- Extract tokens from `choices[0].delta.content`, stop on `data: [DONE]`.
- On mid-stream error: return Err with partial context so the frontend keeps what was generated.
- Use the same pattern as existing `ai.rs` SSE parsing loop.

**providers/anthropic.rs:**
- Same signature as openai_compat but no `provider` param (always Anthropic).
- Build body with top-level `"system"` field (NOT in messages). Include `"max_tokens": 4096, "stream": true, "temperature": 0.1`.
- POST to `https://api.anthropic.com/v1/messages` with headers: `x-api-key: {api_key}` (NOT Authorization: Bearer), `anthropic-version: 2023-06-01`, `Content-Type: application/json`.
- Parse SSE: check `ev.event` field. On `content_block_delta` where `delta.type == "text_delta"`, extract `delta.text`. Stop on `message_stop`. Ignore ping, message_start, content_block_start, content_block_stop, message_delta events.
- Error messages prefixed with "Anthropic: ".

**providers/gemini.rs:**
- Same concept, no `provider` param.
- Translate history: `"assistant"` role -> `"model"` role. Use `contents` array with `parts` sub-arrays. System prompt goes in `systemInstruction.parts[].text`.
- POST to `https://generativelanguage.googleapis.com/v1beta/models/{model}:streamGenerateContent?alt=sse&key={api_key}` -- API key as query param, NOT in header.
- Parse SSE: extract `candidates[0].content.parts[0].text` from each chunk.
- Error messages prefixed with "Google Gemini: ".
