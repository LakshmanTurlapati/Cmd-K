**providers/mod.rs -- add two new Tauri commands:**

`validate_provider_key(provider: String, api_key: String) -> Result<(), String>`:
- Parse provider with `Provider::from_str`.
- Per-provider validation strategy (all use GET on models endpoint to validate key):
  - OpenAI: GET `https://api.openai.com/v1/models` with `Authorization: Bearer {key}`. 200 = valid.
  - Anthropic: GET `https://api.anthropic.com/v1/models` with `x-api-key: {key}` and `anthropic-version: 2023-06-01`. 200 = valid.
  - Gemini: GET `https://generativelanguage.googleapis.com/v1beta/models?key={key}`. 200 = valid.
  - xAI: GET `https://api.x.ai/v1/models` with `Authorization: Bearer {key}`. If 404, fallback to minimal chat completion POST (existing xai.rs pattern). 200 = valid.
  - OpenRouter: GET `https://openrouter.ai/api/v1/models` with `Authorization: Bearer {key}`. 200 = valid.
- On 401: return `Err("{display_name}: Invalid API key. Check your key at {provider_url}.")` where provider_url is the provider's dashboard URL (e.g., "console.anthropic.com", "platform.openai.com", "aistudio.google.com", "console.x.ai", "openrouter.ai/keys").
- On network error: return `Err("{display_name}: Network error. Check your internet connection.")`
- On other HTTP error: return `Err("{display_name}: API error ({status}). Try again.")`

`fetch_provider_models(provider: String, api_key: String) -> Result<Vec<ModelWithMeta>, String>`:
- Parse provider. Read key from keychain (use `get_provider_key` from keychain.rs).
- Wait -- this command receives api_key as param (for validation flow where key isn't saved yet). Use the passed api_key, not keychain.
- For OpenAI, xAI: GET `{base_url}/models` with Bearer auth. Parse `data` array, filter out non-chat models (exclude ids containing "image", "video", "audio", "embedding", "tts", "whisper", "dall-e", "moderation"). Map to `ModelWithMeta` with label from `default_models` if available, otherwise label = "" and tier = "".
- For Anthropic: GET `https://api.anthropic.com/v1/models` with x-api-key + anthropic-version headers. Parse response, map to ModelWithMeta.
- For Gemini: GET `https://generativelanguage.googleapis.com/v1beta/models?key={key}`. Parse `models` array, filter to `generateContent` supported models. Map to ModelWithMeta.
- For OpenRouter: GET `https://openrouter.ai/api/v1/models` with Bearer auth. Parse `data` array. Filter to chat-capable models (those with architecture.modality containing "text->text" or similar). Map to ModelWithMeta.
- Merge with `default_models()`: curated models get their tier tags, API-only models get tier = "" (uncategorized).
- On error: return Err with provider-specific error message.
- If GET /models returns 404 (xAI edge case): return `default_models()` directly.
