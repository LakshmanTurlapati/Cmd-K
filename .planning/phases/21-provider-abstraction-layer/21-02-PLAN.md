    **Create src-tauri/src/commands/models.rs** (replaces xai.rs):

    Define `ModelWithMeta` struct (generalized from `XaiModelWithMeta`):
    ```rust
    #[derive(Serialize, Clone)]
    pub struct ModelWithMeta {
        pub id: String,
        pub label: String,
        pub tier: String,  // "fast", "balanced", "capable", or "" for uncategorized
    }
    ```

    Implement `curated_models(provider: &Provider) -> Vec<ModelWithMeta>` with hardcoded curated model lists per provider (see research for exact model IDs):
    - OpenAI: gpt-4o (Balanced), gpt-4o-mini (Fast), gpt-4.1 (Most Capable), gpt-4.1-mini (Fast), gpt-4.1-nano (Fastest)
    - Anthropic: claude-sonnet-4-20250514 (Balanced), claude-haiku-3-5-20241022 (Fast), claude-opus-4-20250514 (Most Capable)
    - Gemini: gemini-2.0-flash (Fast), gemini-2.5-pro-preview-06-05 (Most Capable), gemini-2.5-flash-preview-05-20 (Balanced)
    - xAI: grok-3 (Balanced), grok-3-mini (Fast), grok-4 (Most Capable)
    - OpenRouter: empty vec (models fetched from API)

    Implement `#[tauri::command] pub async fn validate_api_key(provider: Provider, api_key: String) -> Result<(), String>` with per-provider validation:
    - OpenAI: `GET https://api.openai.com/v1/models` with `Authorization: Bearer {key}` -- 200=valid, 401=invalid_key
    - Anthropic: `POST https://api.anthropic.com/v1/messages` with `x-api-key` + `anthropic-version: 2023-06-01` headers, body `{ "model": "claude-sonnet-4-20250514", "max_tokens": 1, "messages": [{"role":"user","content":"hi"}] }` -- 200=valid, 401=invalid_key
    - Gemini: `GET https://generativelanguage.googleapis.com/v1beta/models?key={key}` -- 200=valid, 400|403=invalid_key
    - xAI: `GET https://api.x.ai/v1/models` with Bearer auth -- 200=valid, 401=invalid_key. If 404, fallback: POST to chat/completions with max_tokens=1 (existing pattern from xai.rs)
    - OpenRouter: `GET https://openrouter.ai/api/v1/models` with Bearer auth -- 200=valid, 401=invalid_key
    - Network errors: return `"Network error: Check your internet connection."`
    - Use `tauri_plugin_http::reqwest::Client::new()` for all requests

    Implement `#[tauri::command] pub async fn fetch_models(provider: Provider, api_key: String) -> Result<Vec<ModelWithMeta>, String>`:
    - Start with curated models for the provider
    - Then fetch from API:
      - OpenAI: `GET /v1/models`, filter to models containing "gpt", map to ModelWithMeta with tier="" for non-curated
      - Anthropic: no public model list API -- return curated only
      - Gemini: `GET /v1beta/models?key={key}`, filter to models supporting `generateContent`, map to ModelWithMeta
      - xAI: `GET /v1/models`, filter out image/video models (existing pattern from xai.rs), map to ModelWithMeta
      - OpenRouter: `GET /api/v1/models`, filter to models where architecture.modality includes "text" or context_length > 0, map to ModelWithMeta with tier=""
    - Merge: curated models first (with tier tags), then API-fetched models not in curated list (with tier="")
    - If API fetch fails, return curated models only (graceful degradation)

    **Update commands/mod.rs:**
    - Replace `pub mod xai;` with `pub mod models;`

    **Update lib.rs:**
    - Replace `xai::validate_and_fetch_models` import with `models::{validate_api_key, fetch_models}`
    - In `invoke_handler`, replace `validate_and_fetch_models` with `validate_api_key, fetch_models` (two separate commands now)
