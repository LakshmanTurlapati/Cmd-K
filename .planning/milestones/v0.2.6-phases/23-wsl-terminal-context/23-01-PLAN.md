```rust
fn is_wsl_ancestor(pid: u32, parent_map: &HashMap<u32, u32>, exe_map: &HashMap<u32, String>) -> bool {
    let mut current = pid;
    for _ in 0..20 {
        if let Some(exe) = exe_map.get(&current) {
            if exe.eq_ignore_ascii_case("wsl.exe") {
                return true;
            }
        }
        match parent_map.get(&current) {
            Some(&ppid) if ppid <= 1 || ppid == current => break,
            Some(&ppid) => current = ppid,
            None => break,
        }
    }
    false
}
```

**4. Linux CWD and shell reading via wsl.exe subprocess (process.rs, Windows path):**

When `is_wsl` is true, the Windows PEB CWD will be a Windows path like `\\wsl.localhost\Ubuntu\home\user`. Instead, read the Linux-native CWD using:

```rust
#[cfg(target_os = "windows")]
fn get_wsl_cwd() -> Option<String> {
    use std::time::Duration;
    let child = std::process::Command::new("wsl.exe")
        .args(["-e", "sh", "-c", "pwd"])
        .stdout(std::process::Stdio::piped())
        .stderr(std::process::Stdio::null())
        .spawn()
        .ok()?;

    // 500ms timeout to avoid hanging if WSL is cold-starting
    let output = child.wait_with_output().ok()?;
    // Note: std::process doesn't have built-in timeout. Use the simpler .output() approach
    // but the entire detect pipeline already has a 750ms outer timeout in detect_full_with_hwnd,
    // so this is bounded by that.
    if output.status.success() {
        let cwd = String::from_utf8_lossy(&output.stdout).trim().to_string();
        if !cwd.is_empty() && cwd.starts_with('/') { Some(cwd) } else { None }
    } else {
        None
    }
}
```

Similarly for shell type:
```rust
#[cfg(target_os = "windows")]
fn get_wsl_shell() -> Option<String> {
    let output = std::process::Command::new("wsl.exe")
        .args(["-e", "sh", "-c", "basename \"$SHELL\""])
        .output()
        .ok()?;
    if output.status.success() {
        let shell = String::from_utf8_lossy(&output.stdout).trim().to_string();
        if !shell.is_empty() { Some(shell) } else { None }
    } else {
        None
    }
}
```

IMPORTANT per RESEARCH Pitfall 6: `wsl.exe -e pwd` returns the HOME directory, not the active shell's CWD. This is acceptable as a baseline. Better CWD comes from UIA text inference (next step).

```rust
fn infer_linux_cwd_from_text(text: &str) -> Option<String> {
    for line in text.lines().rev().take(10) {
        let trimmed = line.trim();
        // Pattern: user@host:/absolute/path$ or user@host:/path #
        if let Some(colon_pos) = trimmed.find(':') {
            if trimmed[..colon_pos].contains('@') {
                let after_colon = &trimmed[colon_pos + 1..];
                let path = after_colon
                    .trim_end_matches(|c: char| c == '$' || c == '#' || c == ' ')
                    .trim();
                if path.starts_with('/') || path == "~" {
                    return Some(path.to_string());
                }
            }
        }
    }
    None
}
```

In `detect_full_with_hwnd`, after UIA text is read and `is_wsl` is true:
- If UIA text yielded a CWD via `infer_linux_cwd_from_text`, use it (overrides the wsl.exe subprocess result).
- If not, keep the wsl.exe subprocess CWD (may be home dir).
- Also infer shell from UIA text using existing `infer_shell_from_text` extended for WSL prompts.

// Linux-specific secrets (WSL terminal context)
// /etc/shadow password hashes ($1$, $5$, $6$, $y$ prefixes)
Regex::new(r"\$[156y]\$[^\s:]+\$[a-zA-Z0-9./+]+").unwrap(),

// SSH private key file detection (the header line, extending existing PEM pattern)
// Already covered by existing PEM header pattern
// Anthropic API keys (sk-ant- prefix)

Be conservative -- do NOT add patterns so broad they redact normal terminal output. The existing patterns are well-scoped. Only add patterns for credentials that are clearly secrets:

1. `/etc/shadow` hash lines: `Regex::new(r"\$[156y]\$[^\s:]+\$[a-zA-Z0-9./+]+")` -- matches password hash format
2. `.env` database/secret URLs: `Regex::new(r"(?i)(DATABASE_URL|REDIS_URL|MONGO_URI)\s*=\s*\S{10,}")`
3. Anthropic API keys: `Regex::new(r"sk-ant-[a-zA-Z0-9_-]{32,}")`
4. Google API keys: `Regex::new(r"AIza[0-9A-Za-z_-]{35}")`
