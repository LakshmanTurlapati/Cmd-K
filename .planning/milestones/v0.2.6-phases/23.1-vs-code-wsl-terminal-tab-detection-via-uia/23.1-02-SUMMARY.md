## Accomplishments
- Removed wsl.exe from KNOWN_TERMINAL_EXES to prevent VS Code's 10-16+ internal wsl.exe processes from polluting shell candidate search
- Added IDE-aware shell priority in find_shell_by_ancestry that prefers interactive shells (powershell, pwsh, bash, zsh, fish) over cmd.exe for IDE terminals
- Non-IDE terminals (Windows Terminal, standalone PowerShell) are completely unaffected by the new logic

## Decisions Made
- Deprioritize cmd.exe rather than exclude it -- when only cmd.exe descendants exist in an IDE, it still gets selected as the shell
- Removed wsl.exe from KNOWN_TERMINAL_EXES entirely rather than adding special-case logic -- WSL detection already handled by window title patterns and UIA text analysis from plan 01
- Interactive shell preference applies only to IDE terminals (is_ide_with_terminal_exe check), preserving existing behavior for all non-IDE terminal emulators
