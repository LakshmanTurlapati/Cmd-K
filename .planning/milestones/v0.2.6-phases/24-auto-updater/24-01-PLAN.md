4. **state.rs**: Add an `UpdateState` struct with an enum for update lifecycle:
   ```rust
   #[derive(Debug, Clone)]
   pub enum UpdateStatus {
       Idle,                    // "Check for Updates..."
       Checking,                // "Checking for Updates..."
       Available(String),       // "Update Available (vX.Y.Z)"
       Downloading(String),     // "Downloading vX.Y.Z..."
       Ready(String),           // "Update Ready (restart to apply)"
       Disabled,                // Auto-check disabled by user
   }

   pub struct UpdateState {
       pub status: Mutex<UpdateStatus>,
       pub pending_update: Mutex<Option<tauri_plugin_updater::Update>>,
       pub pending_bytes: Mutex<Option<Vec<u8>>>,
   }
   ```
   Add `UpdateState` as a separate managed state (NOT inside AppState -- use `app.manage(UpdateState::new())` separately, because `tauri_plugin_updater::Update` is not `Default`).

5. **commands/mod.rs**: Add `pub mod updater;`

6. **commands/updater.rs**: Create the update lifecycle module:
   - `UpdateState::new()` constructor returning `Idle` status, `None` for pending fields.
   - `pub fn spawn_update_checker(app_handle: tauri::AppHandle)`: Uses `tauri::async_runtime::spawn` to:
     a. Call `check_and_download(&app_handle).await` immediately (launch check).
     b. Enter a `tokio::time::interval(Duration::from_secs(86400))` loop, calling `check_and_download` each tick.
   - `async fn check_and_download(app: &tauri::AppHandle)`:
     a. Read auto-update preference from `tauri-plugin-store` (key: `"autoUpdate"` in `settings.json`). If `false`, set status to `Disabled` and return early. Default to `true` if key is missing.
     b. Set status to `Checking`, update tray text.
     c. Call `app.updater()?.check().await`. On `Ok(Some(update))`:
        - Set status to `Available(version)`, update tray text.
        - Immediately call `update.download(|_,_| {}, || {}).await` to auto-download.
        - On download success: store bytes in `pending_bytes`, store update in `pending_update`, set status to `Ready(version)`, update tray text.
        - On download failure: reset status to `Idle`, update tray text (silent failure per CONTEXT.md).
     d. On `Ok(None)`: set status to `Idle`, update tray text.
     e. On `Err(_)`: set status to `Idle`, update tray text (silent failure).
   - `pub fn update_tray_text(app: &tauri::AppHandle, status: &UpdateStatus)`: Get tray by id `"main"`, get its menu, find item by id `"check_for_updates"`, call `set_text()` with:
     - `Idle` -> "Check for Updates..."
     - `Checking` -> "Checking for Updates..."
     - `Available(v)` -> "Update Available (v{v})"  (no emoji -- per CONTEXT.md uses unicode arrow, but let's use plain text for cross-platform compat. Actually CONTEXT.md says "Update Available (vX.Y.Z)" without emoji for the menu text)
     - `Downloading(v)` -> "Downloading v{v}..."
     - `Ready(v)` -> "Update Ready (restart to apply)"
     - `Disabled` -> "Check for Updates..."
   - `pub fn install_pending_update(app: &tauri::AppHandle)`: Takes `pending_bytes` and `pending_update` from `UpdateState`. If both exist, calls `update.install(bytes)`. This is called from the quit handler.
   - `pub async fn manual_update_check(app: tauri::AppHandle)`: Triggered when user clicks "Check for Updates..." menu item. Calls `check_and_download(&app).await`.

   **IMPORTANT**: Use `tauri_plugin_updater::UpdaterExt` trait for `app.updater()`. The `Update` type from the plugin stores download bytes. Use `Arc<Mutex<>>` patterns consistent with the codebase (plain `std::sync::Mutex` like AppState).

   **IMPORTANT**: The `update.download()` callback signature is `download(on_chunk: F, on_download_finish: G)` where `F: FnMut(usize, Option<u64>)` and `G: FnOnce()`. Use closures that do nothing for the simple case.

   **IMPORTANT**: For `update_tray_text`, use `app.tray_by_id("main")` to get the tray icon, then access its menu. The menu item ID is `"check_for_updates"`. To find and update a specific menu item, iterate or use `menu.get("check_for_updates")` -- check the Tauri v2 Menu API. If `menu.get()` is not available, store the `MenuItem` reference in `UpdateState` or use `app.menu().get()`. The simplest approach: use `MenuItem::with_id` in tray.rs and then retrieve via the tray menu's `items()` method, or use `app.try_state::<MenuItem>()`. **Actually**, the cleanest Tauri v2 pattern is: the `MenuItem` created in `setup_tray` should be stored somewhere accessible. Add the `MenuItem` for updates to `UpdateState` as `pub menu_item: Mutex<Option<tauri::menu::MenuItem<tauri::Wry>>>`. Set it during `setup_tray`. Then `update_tray_text` reads it from state.

7. **lib.rs**:

   - At the END of `setup()` (after `window.hide().ok()` but before `Ok(())`), add:

