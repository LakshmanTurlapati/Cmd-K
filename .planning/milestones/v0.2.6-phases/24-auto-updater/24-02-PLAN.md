Modify `.github/workflows/release.yml` to support updater artifact generation.

**macOS build job (`build-macos`)**:

1. Add `TAURI_SIGNING_PRIVATE_KEY` and `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` env vars to the "Build signed and notarized DMG" step:
   ```yaml
   env:
     # ... existing env vars ...
     TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
     TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
   ```
   When these env vars are set and `createUpdaterArtifacts: true` is in tauri.conf.json, `pnpm tauri build` (called by build-dmg.sh) automatically generates:
   - `target/universal-apple-darwin/release/bundle/macos/CMD+K.app.tar.gz`
   - `target/universal-apple-darwin/release/bundle/macos/CMD+K.app.tar.gz.sig`

2. Add a step AFTER "Generate checksum" to copy updater artifacts to the release directory:
   ```yaml
   - name: Copy updater artifacts
     run: |
       UPDATER_DIR="src-tauri/target/universal-apple-darwin/release/bundle/macos"
       cp "$UPDATER_DIR"/CMD+K.app.tar.gz release/ 2>/dev/null || true
       cp "$UPDATER_DIR"/CMD+K.app.tar.gz.sig release/ 2>/dev/null || true
       # Verify the files exist
       ls -la release/CMD+K.app.tar.gz*
   ```

3. Update the "Upload artifacts" step path to include the new files. The existing glob `release/CMD+K-*.dmg*` won't match `.tar.gz`. Change to:
   ```yaml
   path: |
     release/CMD+K-*.dmg*
     release/CMD+K.app.tar.gz
     release/CMD+K.app.tar.gz.sig
   ```

**Windows build job (`build-windows`)**:

1. Add `TAURI_SIGNING_PRIVATE_KEY` and `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` env vars to the "Build Tauri app" step:
   ```yaml
   - name: Build Tauri app
     env:
       TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
       TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
     run: pnpm tauri build
   ```
   This generates a `.exe.sig` file alongside the NSIS installer in `src-tauri/target/release/bundle/nsis/`.

2. **IMPORTANT**: The existing "Rename artifact" step renames the .exe but NOT the .sig. The .sig is generated for the ORIGINAL filename. We need to read the .sig BEFORE renaming, or copy the .sig alongside the renamed .exe. The simplest approach: after renaming the .exe, also rename the .sig:
   ```yaml
   - name: Rename artifact
     shell: bash
     run: |
       NSIS_DIR="src-tauri/target/release/bundle/nsis"
       ORIG_EXE=$(ls "$NSIS_DIR"/*.exe | grep -v '.exe.sig' | head -1)
       ORIG_SIG="${ORIG_EXE}.sig"
       mv "$ORIG_EXE" "$NSIS_DIR/CMD+K-${{ env.VERSION }}-windows-x64.exe"
       if [ -f "$ORIG_SIG" ]; then
         mv "$ORIG_SIG" "$NSIS_DIR/CMD+K-${{ env.VERSION }}-windows-x64.exe.sig"
       fi
   ```

3. The upload artifacts path `src-tauri/target/release/bundle/nsis/CMD+K-*.*` should already catch both .exe and .exe.sig. Verify this glob matches. If not, update to:
   ```yaml
   path: |
     src-tauri/target/release/bundle/nsis/CMD+K-${{ env.VERSION }}*
   ```

**Release job (`release`)**:

1. Add a step BEFORE "Create GitHub Release" to assemble `latest.json`:
   ```yaml
   - name: Assemble latest.json
     run: |
       VERSION="${{ env.VERSION }}"
       REPO_URL="https://github.com/LakshmanTurlapati/Cmd-K/releases/download/v${VERSION}"

       # Read signatures
       SIG_MACOS=""
       if [ -f "artifacts/macos-dmg/CMD+K.app.tar.gz.sig" ]; then
         SIG_MACOS=$(cat "artifacts/macos-dmg/CMD+K.app.tar.gz.sig")
       fi

       SIG_WINDOWS=""
       if [ -f "artifacts/windows-installer/CMD+K-${VERSION}-windows-x64.exe.sig" ]; then
         SIG_WINDOWS=$(cat "artifacts/windows-installer/CMD+K-${VERSION}-windows-x64.exe.sig")
       fi

       cat > artifacts/latest.json <<EOF
       {
         "version": "${VERSION}",
         "pub_date": "$(date -u +%Y-%m-%dT%H:%M:%SZ)",
         "platforms": {
           "darwin-aarch64": {
             "signature": "${SIG_MACOS}",
             "url": "${REPO_URL}/CMD+K.app.tar.gz"
           },
           "darwin-x86_64": {
             "signature": "${SIG_MACOS}",
             "url": "${REPO_URL}/CMD+K.app.tar.gz"
           },
           "windows-x86_64": {
             "signature": "${SIG_WINDOWS}",
             "url": "${REPO_URL}/CMD+K-${VERSION}-windows-x64.exe"
           }
         }
       }
       EOF

       echo "latest.json contents:"
       cat artifacts/latest.json
   ```

   **IMPORTANT platform keys**: Use `darwin-aarch64` and `darwin-x86_64` (both pointing to the same universal .app.tar.gz). Use `windows-x86_64` for Windows. These must match what the Tauri updater expects based on the target architecture.

2. Update the "Create GitHub Release" `files` section to include updater artifacts:
   ```yaml
   files: |
     artifacts/macos-dmg/CMD+K-*.dmg*
     artifacts/macos-dmg/CMD+K.app.tar.gz
     artifacts/macos-dmg/CMD+K.app.tar.gz.sig
     artifacts/windows-installer/CMD+K-*.*
     artifacts/latest.json
   ```

3. Update the release body to mention auto-update support:

   ```yaml
     ### Auto-Update

     Users on v0.2.6+ will receive this update automatically.

     ### Checksums (SHA256)

     See the `.sha256` files attached to this release.

     ---
     _Edit this release to add detailed release notes._
   ```
