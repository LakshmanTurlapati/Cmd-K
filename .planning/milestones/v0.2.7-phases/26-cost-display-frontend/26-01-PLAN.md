In src-tauri/src/commands/usage.rs:
- Add `pub query_costs: Vec<Option<f64>>` to UsageStatsResponse
- The query_costs are tracked per-query at recording time. Since the cost calculation happens at read time (get_usage_stats), we need a different approach: store per-query metadata in the accumulator so costs can be calculated later.

Revised approach for per-query cost tracking:
- In state.rs, add a struct `QueryRecord { provider: String, model: String, input_tokens: u64, output_tokens: u64 }` and add `query_history: Vec<QueryRecord>` to UsageAccumulator
- In `record()`, push a QueryRecord with the actual token values (skip if both are None)
- In `reset()`, clear query_history
- In usage.rs `get_usage_stats()`, iterate query_history to compute query_costs: for each QueryRecord, look up pricing (same curated then OpenRouter logic), calculate cost or None, collect into Vec<Option<f64>>
- Add `query_costs: Vec<Option<f64>>` to UsageStatsResponse and populate it

This keeps cost calculation centralized in usage.rs where pricing data is available.
