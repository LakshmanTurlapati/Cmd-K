<task type="auto">
  <name>Task 2: Extract token usage from all 3 adapters and wire accumulation in ai.rs</name>
  <files>src-tauri/src/commands/providers/openai_compat.rs, src-tauri/src/commands/providers/anthropic.rs, src-tauri/src/commands/providers/gemini.rs, src-tauri/src/commands/ai.rs</files>
  <action>
**Change adapter return types** from `Result<(), String>` to `Result<TokenUsage, String>` in all three adapters. Import `TokenUsage` from `crate::state::TokenUsage`.

**openai_compat.rs (TRAK-01):**

1. Add `"stream_options": {"include_usage": true}` to the JSON request body alongside `"stream": true`. This tells OpenAI/xAI/OpenRouter to include a final chunk with usage data. Providers that don't support it ignore the unknown field.

2. Add a mutable `let mut token_usage = TokenUsage::default();` before the stream loop.

3. In the stream parsing loop, after the existing `choices[0].delta.content` extraction, also check for the usage chunk: when `chunk["usage"]` exists (the final chunk has `choices: []` and `usage` object), extract `chunk["usage"]["prompt_tokens"].as_u64()` into `token_usage.input_tokens` and `chunk["usage"]["completion_tokens"].as_u64()` into `token_usage.output_tokens`.

4. Return `Ok(token_usage)` instead of `Ok(())`.

**anthropic.rs (TRAK-02):**

1. Add a mutable `let mut token_usage = TokenUsage::default();` before the stream loop.

2. In the event match, add handling for `"message_start"` event: parse `chunk["message"]["usage"]["input_tokens"].as_u64()` into `token_usage.input_tokens`.

3. Add handling for `"message_delta"` event: parse `chunk["usage"]["output_tokens"].as_u64()` into `token_usage.output_tokens`.

4. Return `Ok(token_usage)` instead of `Ok(())`.

**gemini.rs (TRAK-03):**

1. Add a mutable `let mut token_usage = TokenUsage::default();` before the stream loop.

2. Inside the chunk parsing (after existing text extraction), check for `chunk["usageMetadata"]`. If present, extract `chunk["usageMetadata"]["promptTokenCount"].as_u64()` into `token_usage.input_tokens` and `chunk["usageMetadata"]["candidatesTokenCount"].as_u64()` into `token_usage.output_tokens`. Note: Gemini may send usageMetadata on multiple chunks -- always overwrite with latest values (the last chunk has final counts).

3. Return `Ok(token_usage)` instead of `Ok(())`.

**ai.rs (TRAK-04 -- accumulation):**

1. Add `tauri::State` parameter to `stream_ai_response`: `state: tauri::State<'_, crate::state::AppState>`.

2. After the `match provider.adapter_kind()` block (which now returns `Result<TokenUsage, String>`), capture the result: `let token_usage = match provider.adapter_kind() { ... }?;`

3. After successful token_usage return, accumulate:
   ```rust
   if let Ok(mut acc) = state.usage.lock() {
       acc.record(provider.display_name(), &model, &token_usage);
   }
   ```

4. Return `Ok(())` from the command (the IPC return type stays `Result<(), String>` -- frontend doesn't need token_usage directly).

**Important:** When a provider doesn't return token counts (e.g., stream_options not supported), TokenUsage fields will be None -- this is correct. The accumulator's `record()` method skips None values, so no tokens are added for that query. No warnings, no errors.
