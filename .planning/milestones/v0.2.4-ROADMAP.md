# Milestone v0.2.4: Overlay UX, Safety & CI/CD

**Status:** SHIPPED 2026-03-04
**Phases:** 17-20
**Total Plans:** 5

## Overview

Overlay UX polish (z-order fix so system dialogs appear above overlay, drag-to-reposition with session-scoped memory), comprehensive cross-platform destructive command safety (150 patterns), Windows Terminal shell detection fix, and fully automated CI/CD release pipeline via GitHub Actions.

## Phases

### Phase 17: Overlay Z-Order

**Goal**: System UI elements (permission dialogs, Notification Center, Spotlight) can appear above the CMD+K overlay while the overlay still floats above normal application windows
**Depends on**: Nothing (first phase of v0.2.4)
**Requirements**: ZORD-01, ZORD-02
**Plans**: 1 plan

Plans:
- [x] 17-01-PLAN.md -- Lower NSPanel level from Status to Floating, verify z-order with system UI

**Details:**
Changed NSPanel window level from PanelLevel::Status (25) to PanelLevel::Floating (3) in src-tauri/src/lib.rs. Floating is the standard macOS level for utility panels (same as Raycast). Fullscreen support preserved via full_screen_auxiliary() collection behavior.

### Phase 18: Draggable Overlay Positioning

**Goal**: User can reposition the overlay by dragging it, and the overlay reopens at the last dragged position until the app is relaunched
**Depends on**: Phase 17
**Requirements**: OPOS-01, OPOS-02, OPOS-03
**Plans**: 1 plan

Plans:
- [x] 18-01-PLAN.md -- Add drag-to-reposition with in-memory position persistence and useDrag hook

**Details:**
Created useDrag React hook with screen-coordinate delta tracking, attached to App.tsx panel. Position stored in Rust AppState as Mutex<Option<(f64, f64)>> (session-scoped, resets on relaunch). set_overlay_position IPC command persists drag position. 2px dead zone prevents accidental position changes from clicks. Interactive elements excluded via target.closest() selector.

### Phase 19: Enhance Destructive Commands (Cross-Platform)

**Goal**: Expand destructive command regex patterns from ~80 to 150+ covering macOS, Linux, Windows, containers, package managers, and config file overwrites
**Depends on**: Phase 18
**Requirements**: SAFE-01, SAFE-02, SAFE-03, SAFE-04, SAFE-05, SAFE-06
**Plans**: 1 plan

Plans:
- [x] 19-01-PLAN.md -- Add exhaustive cross-platform destructive patterns

**Details:**
Expanded DESTRUCTIVE_PATTERNS in safety.rs from ~80 to 150 patterns across 10 sections. Added macOS (csrutil, dscl, nvram, etc.), Linux (systemctl, iptables, userdel, etc.), container/orchestration (docker, kubectl, helm, terraform, vagrant), package manager (apt, brew, pip, npm, cargo, choco, etc.), and config file overwrite patterns. All organized with // === Section === headers.

### Phase 20: GitHub Actions CI/CD Pipeline

**Goal**: Automated CI/CD pipeline that builds signed+notarized macOS DMG and unsigned Windows NSIS installer on v* tag push
**Depends on**: Phase 19
**Requirements**: CICD-01, CICD-02, CICD-03, CICD-04, CICD-05, CICD-06
**Plans**: 2 plans

Plans:
- [x] 20-01-PLAN.md -- Parameterize build-dmg.sh for CI + credential migration guide
- [x] 20-02-PLAN.md -- Create release.yml workflow with macOS/Windows builds and GitHub Release publishing

**Details:**
Plan 01: Parameterized build-dmg.sh with env var fallbacks (VERSION, SIGNING_IDENTITY, KEYCHAIN_PROFILE). APPLE_ID presence determines CI vs local notarization path. CI guard skips DMG Finder styling. Created docs/ci-secrets-setup.md.
Plan 02: Created .github/workflows/release.yml with 3-job architecture (build-macos, build-windows, release). macOS imports p12 into temp keychain, stores notarization creds, runs build-dmg.sh. Windows builds NSIS with conditional signing. Release job auto-publishes via softprops/action-gh-release@v2 with SHA256 checksums.

## Milestone Summary

**Key Decisions:**
- PanelLevel::Floating (3) instead of ModalPanel (8) for utility panels
- In-memory Mutex position state -- no disk persistence
- Screen coordinates for drag deltas (window moves during drag)
- 2px dead zone before persisting position
- // === Section === format for destructive pattern categories
- grep+sed for version extraction (no jq dependency)
- APPLE_ID env var presence determines CI vs local notarization
- 3-job CI architecture for parallel builds + sequential publish
- Conditional Windows signing gated on secret presence

**Issues Resolved:**
- NSPanel Status window level blocking system overlays (Phase 17)
- Windows Terminal ConPTY shell type detection failure (Phase 19 merge)

**Issues Deferred:**
- Windows OV/EV code signing (purchase when distribution warrants it)
- PR build verification (compile-only, no artifacts)
- Auto-updater (tauri-plugin-updater)
- Linux builds

**Technical Debt Incurred:**
- No automated tests for destructive command patterns (manual verification only)
- DMG Finder window styling skipped in CI (functional but plain layout)
