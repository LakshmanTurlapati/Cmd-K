  <name>Task 1: Refactor detection functions to use shared ProcessSnapshot</name>
  <files>src-tauri/src/terminal/process.rs</files>
  <action>
Refactor the three functions that currently take their own snapshots to accept a shared ProcessSnapshot reference instead. This is the core PROC-03 consolidation.

1. **Refactor find_shell_by_ancestry** (lines 767-896):
   - Change signature to `fn find_shell_by_ancestry(app_pid: i32, _focused_cwd: Option<&str>, snapshot: &ProcessSnapshot) -> Option<i32>`
   - Remove the internal CreateToolhelp32Snapshot call, parent_map/shell_candidates/openconsole_pids construction (lines 774-808). Use `snapshot.parent_map`, `snapshot.shell_candidates`, `snapshot.conpty_host_pids` instead.
   - **ConPTY-first selection (PROC-01):** Change the detection priority:
     a. FIRST: Find shells whose parent is in snapshot.conpty_host_pids (ConPTY-hosted shells). These are interactive terminal shells regardless of whether they're descendants of app_pid.
     b. Among ConPTY shells, filter to those that are descendants of app_pid (walk parent chain). If the app is a known IDE or terminal, ConPTY shells may NOT be direct descendants (ConPTY architecture) -- in that case, accept all ConPTY shells as candidates.
     c. SECOND: Direct descendant walk (existing behavior) for standalone terminal scenarios.
     d. THIRD: Highest-PID fallback for older systems without ConPTY hosts.
   - **cmd.exe filtering (PROC-02 integration):** Replace the blanket cmd.exe exclusion block (lines 865-889) with `is_interactive_cmd()` filtering. For each cmd.exe candidate, call `is_interactive_cmd(pid, snapshot)`. If it returns false, exclude it. If true, keep it as a valid shell candidate. This means cmd.exe users in IDEs are now supported.
   - **Recency sorting:** Replace `max_by_key(|(pid, _)| *pid)` with GetProcessTimes-based sorting. Add a helper `get_process_creation_time(pid: u32) -> Option<u64>` that calls GetProcessTimes and returns CreationTime as a u64 for comparison. Sort candidates by creation time descending (most recent first). Fall back to PID comparison if GetProcessTimes fails.

2. **Refactor detect_wsl_in_ancestry** (lines 1024-1075):
   - Change signature to `fn detect_wsl_in_ancestry(pid: i32, snapshot: &ProcessSnapshot) -> bool`
   - Remove the internal CreateToolhelp32Snapshot call and map construction (lines 1031-1056). Use `snapshot.parent_map` and `snapshot.exe_map` directly.
   - Keep the ancestry walk logic (lines 1058-1073) unchanged, just reading from snapshot.

3. **Refactor scan_wsl_processes_diagnostic** (lines 964-1017):
   - Change signature to `fn scan_wsl_processes_diagnostic(_shell_pid: i32, snapshot: &ProcessSnapshot)`
   - Remove the internal CreateToolhelp32Snapshot call and map construction (lines 971-997). Build the wsl_pids list by iterating `snapshot.exe_map` instead.
   - Keep the parent chain logging logic unchanged, reading from snapshot.

4. **Update get_foreground_info** (lines 89-165):
   - Change signature to accept an optional snapshot: `pub fn get_foreground_info(terminal_pid: i32, snapshot: Option<&ProcessSnapshot>) -> ProcessInfo`
   - On Windows: if snapshot is provided, pass it to find_shell_by_ancestry (via find_shell_pid), detect_wsl_in_ancestry, and scan_wsl_processes_diagnostic. If None, create one internally as fallback.
   - Update find_shell_pid to accept and pass through the snapshot.
   - On non-Windows: ignore the snapshot parameter.

5. **Update find_shell_pid**:
   - Change Windows signature to `fn find_shell_pid(terminal_pid: i32, focused_cwd: Option<&str>, snapshot: Option<&ProcessSnapshot>) -> Option<i32>`
   - Pass snapshot to find_shell_by_ancestry.
   - Non-Windows signature unchanged (snapshot parameter ignored or absent via cfg).

All changes MUST be `#[cfg(target_os = "windows")]` gated. The macOS find_shell_by_ancestry and any non-Windows code paths must remain completely untouched.
