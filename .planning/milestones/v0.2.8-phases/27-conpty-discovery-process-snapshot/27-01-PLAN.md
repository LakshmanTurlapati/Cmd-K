Add the following to process.rs, all gated with #[cfg(target_os = "windows")] except the pure string parsing functions which should be cross-platform testable:

1. **ProcessSnapshot struct** (pub(crate) visibility):
   ```rust
   pub(crate) struct ProcessSnapshot {
       pub parent_map: std::collections::HashMap<u32, u32>,
       pub exe_map: std::collections::HashMap<u32, String>,
       pub shell_candidates: Vec<(u32, String)>,
       pub conpty_host_pids: Vec<u32>,
   }
   ```
   Add `ProcessSnapshot::capture() -> Option<Self>` that does a single CreateToolhelp32Snapshot call, iterates all processes, and populates all four fields. Check for BOTH "OpenConsole.exe" AND "conhost.exe" (case-insensitive) as ConPTY hosts -- the existing code only checks OpenConsole.exe which misses Windows 10. Use `super::detect_windows::is_known_shell_exe()` to identify shell candidates. Log: `eprintln!("[process] ProcessSnapshot captured: {} processes, {} shells, {} ConPTY hosts", ...)`.

2. **read_command_line_from_peb** (Windows-only, unsafe fn):
   Extend the existing `read_cwd_from_peb` pattern. Read UNICODE_STRING at `params_ptr + 0x70` (CommandLine) instead of `params_ptr + 0x38` (CurrentDirectory). Return `Option<String>` with the full command line. On access-denied or read failure, return None gracefully with eprintln logging. Do NOT crash on elevated processes.

3. **extract_cmd_args** (cross-platform, pub for testing):
   Parse a full command line string to extract everything after the exe path. Handle both quoted paths (`"C:\Program Files\cmd.exe" /C dir`) and unquoted paths (`cmd.exe /C dir`). Return the args portion as a &str.

4. **has_batch_flag_in_cmdline** (cross-platform, pub for testing):
   Takes a full command line string. Calls extract_cmd_args internally. Checks if the args start with (case-insensitive) `/C`, `/D /C`, `/S /D /C`, or `/C"`. Returns true if cmd.exe was launched for batch execution.

5. **is_interactive_cmd** (Windows-only):
   Takes pid and &ProcessSnapshot. Two-signal approach per CONTEXT.md locked decision:
   - FAST CHECK: Is parent (from snapshot.parent_map) in snapshot.conpty_host_pids? If yes -> interactive (return true).
   - If parent is node.exe, Code.exe, Code Helper, Cursor.exe (case-insensitive contains check) -> likely internal, do definitive check.
   - DEFINITIVE CHECK: OpenProcess the pid, call read_command_line_from_peb. If command line has batch flags -> not interactive (return false). If no batch flags or PEB read fails -> treat as interactive (return true, conservative fallback).
   - If PEB read fails due to access denied, log and fall back to parent-based heuristic only.

6. **Unit tests** in a `#[cfg(test)] mod tests` block at the bottom of process.rs:
   - Test extract_cmd_args with quoted paths, unquoted paths, bare exe name.
   - Test has_batch_flag_in_cmdline with all the behavior cases listed above.
   - These tests are pure string parsing and run on all platforms (no #[cfg(windows)] needed).
